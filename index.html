02 — The Vision

Moving work upstream

The Core Shift: From "Engineer Builds" to "Designer Drafts with AI, Engineer Reviews".

Future workflow

How we will work tomorrow

Designer

Uses Figma + AI Prompts

Claude Code · HeroGen

AI

Generates Draft Code

Storybook · SwiftUI · Compose

Engineer

Reviews and approves

By leveraging AI, Flamingo designers can generate the initial code drafts directly from Figma, leaving engineers to focus on review, refinement, and architecture — the parts that truly need their expertise.

This shift is ideal for uncomplicated tasks like component or token updates. Intricate projects (e.g. building Dark Mode from scratch) will remain engineer-led.

03 — Q3 Roadmap

Step-by-step implementation

Strategy: To ensure success, we will tackle these proposals one platform at a time, starting with Web. Building a complete end-to-end flow first lets us learn and iterate before expanding to iOS and Android.

Goal: Allow Flamingo designers to create and update components using HeroGen and Claude Code, eliminating the need to create three separate Jira tickets.

  • Map out the exact technical workflow used by engineering today.
  • Connect our code repositories to AI tools (Claude Code & HeroGen).
  • Test the pipeline by building or updating test components.

Goal: Ensure our component galleries stay in sync automatically as designers become more involved in the development stage.

  • Figure out how to securely connect Storybook to Claude Code / HeroGen.
  • Run a test update on a single component to ensure the gallery updates on release.

Goal: Create a definitive guide explaining the capabilities, setup, and best use cases for each tool based on our Q3 tests.

  • Execute Proposals 1 and 2 using both tools.
  • Document the pros & cons and build a "When to Use What" matrix.

The Issue: Our current Zeroheight plan lacks MCP (Model Context Protocol) support, making agentic AI integration impossible.

  • Extract all current content and component web previews from Zeroheight.
  • Build the new AI-supported documentation platform.
  • Run AI audits to flag inconsistencies and draft fixes.
04 — Future Horizons

Q4 & Beyond

Once our foundational AI pipeline is solid, we can automate even more of our ecosystem.

🤖

AI Documentation Agents

Automatically update the documentation site the moment a component's code changes — no manual copy-paste required.

🔄

Automated Token Sync

A designer publishes in Figma → a webhook triggers → CI runs → Pull Requests open across Web, iOS, and Android simultaneously for engineering approval.

🎨

Illustration Agents

Use AI to generate, format, and store asset variants automatically — keeping our illustration library consistent and up to date.

👁

Design Critique Agents

AI bots that review Figma files against our documented design tokens and guidelines — catching errors before a single line of code is written.
